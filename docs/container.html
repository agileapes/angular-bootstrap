Containers

Containers can serve to compartmentalize your content, hiding away those that do not require immediate attention.

Containers can be presented in two ways: tabs and pills.

Tabs

Tabs are generally used to indicate main sections of an application.

Example

This is my home Here goes the profile. This is the tab that is active by default. Some messages might be here And this is the settings
HTML
<ui:container type="tab">
    <ui:section title="My Home" glyph="home">
        This is my home
    </ui:section>
    <ui:section glyph="paperclip" active="true" title="Profile">
        Here goes the profile. This is the tab that is <strong>active</strong> by default.
    </ui:section>
    <ui:section glyph="envelope" title="Messages">Some messages might be here</ui:section>
    <ui:section glyph="wrench" title="Settings">And this is the settings</ui:section>
    <ui:section glyph="qrcode" title="Random"></ui:section>
</ui:container>
JavaScript
<script type="text/javascript"><!--
    $(function () {
        $("ui\\:container").on("changed", function (event, state) {
            if (state.sections()[state.to].title.toLowerCase() == "random") {
                state.sections()[state.to].$content.html(Math.random());
            }
        });
    });
//--></script>

Pills

Using pills, you can separate minor sections of your logic from each other.

Example

This is the first one This is the second one 
<ui:container type="pill">
    <ui:section title="First" active="true">This is the first one</ui:section>
    <ui:section title="Second">This is the second one</ui:section>
</ui:container>
